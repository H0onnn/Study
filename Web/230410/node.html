- Node.js란? 1. Node.js는 Chrome V8 자바스크립트 엔진으로 빌드된 JavaScript 런타임. 2. Node.js는 서버 측 즉, 브라우저의 외부에서 JavaScript를 실행할 수 있도록 하는 것을 주된 목적으로 함. 3. Node.js는 이벤트 기반, 논블로킹 I/O 모델(비동기 처리 모델)을 사용하여 가볍고 효율적인 네트워크 앱을 만들 수 있다. - 모듈과 Exports 1. Node.js에서 모듈은 코드의 재사용을 위한 단위를 뜻한다. 2. Node.js의 모듈 시스템은 CommonJS 스펙을 따른다. 3. 모듈에서 함수, 객체, 변수 등을 외부에 노출하려면 module.exports 객체를 사용한다. 4. exports 객체는 module.exports의 참조로 사용되며, 외부에 노출하려는 함수, 객체, 변수를 exports 객체에 직접 추가할 수 있다. (아래 예시 코드를 살펴보자) 사용법 -1 var 함수 = function(){ console.log("반가워!") }; module.exports = { 함수 } 사용법 -2 module.exports = { 인사 : () => console.log("반가워!") } 사용법 -3 exports.반가워 = () => console.log("반가워!") ** CommonJS와 Module의 차이점 ** 1. Node.js에서 사용되는 모듈 시스템 2. 동기적으로 로드됨 3. require() 함수 사용 4. module.exports와 exports 객체 사용 가능 1. 웹 브라우저에서 사용되는 모듈 시스템 2. 비동기적으로 로드됨 3. import/export 키워드 사용 4. export 키워드를 사용하여 노출할 항목 지정 - npm과 package.json 1. npm(Node Package Manager의 약자)은 Node.js 패키지 관리자를 뜻한다. 2. 패키지는 Node.js에서 사용되는 모듈을 포함하는 디렉토리를 의미한다. 3. npm install 명령어를 사용하여 패키지를 설치하고 require() 함수를 사용하여 패키지를 로드할 수 있다. 4. package.json 파일은 프로젝트의 정보와 의존성 모듈 정보를 담고 있다.
